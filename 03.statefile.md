## terraform.tfstate

> `terraform.tfstate` → Used to **track resources**

⚠️ **Should NOT be updated manually in any case.**

| # | What it Stores |
|---|----------------|
| i | What exists |
| ii | Resource IDs |
| iii | Current state |
| iv | Mapping between code & real cloud |

- JSON file
- Each folder has its **own** `.tf` files and `terraform.tfstate`
- Tracks **only** the resources created/applied from that specific project

```
terraform-ec2/  →  creates EC2  →  state file tracks only EC2
terraform-s3/   →  creates S3   →  state file tracks only S3
```

### ❌ Where NOT to Store State File

| Location | Problem |
|----------|---------|
| GitHub / Source Control | Contains sensitive info about cloud resources |
| Local machine | Multiple people work on the project; local state can't be shared |

✅ **Solution → Store Remotely (Remote Backend)**
